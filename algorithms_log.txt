                         VIDYA ADAPTIVE ALGORITHMS LOG
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This document details the mathematical models and logic rules implementing the
personalized learning engine of the Vidya application. All logic operates fully
offline using local state.

=============================================================================
1. BAYESIAN KNOWLEDGE TRACING (BKT)
=============================================================================
Used to estimate the probability that a student has mastered a specific concept
based on their sequence of quiz responses.

Parameters:
  - P(L0): Prior probability the student knows the concept (Default = 0.25)
  - P(T) : Probability of transition (learning) on a study attempt (Default = 0.20)
  - P(G) : Probability of guessing correctly without knowing (Default = 0.20)
  - P(S) : Probability of slipping (incorrect answer despite knowing) (Default = 0.10)

Bayesian Update Equations:
  When the student answers CORRECTLY (evidence = correct):
    P(Ln | correct) = P(Ln-1) * (1 - P(S)) / [P(Ln-1) * (1 - P(S)) + (1 - P(Ln-1)) * P(G)]

  When the student answers INCORRECTLY (evidence = wrong):
    P(Ln | wrong) = P(Ln-1) * P(S) / [P(Ln-1) * P(S) + (1 - P(Ln-1)) * (1 - P(G))]

Transition Update (Learning):
  P(Ln+1) = P(Ln | evidence) + (1 - P(Ln | evidence)) * P(T)

Mastery State:
  A concept is considered "Mastered" when P(Ln+1) >= 0.85.


=============================================================================
2. SPACED REPETITION SYSTEM (SRS) — SUPERMEMO-2 (SM-2)
=============================================================================
Determines the optimal next review date for each flashcard.

Variables:
  - EF (Easiness Factor): Default starting value is 2.5. Minimum value is 1.3.
  - Repetition Count (r): Number of consecutive successful reviews.
  - Interval (I): Time in days until the next scheduled review.

Difficulty Rating (q) entered by student (0 to 5):
  5: Perfect response, complete recall
  4: Correct response after hesitation
  3: Correct response with serious difficulty
  2: Incorrect response, but easy to recall upon reveal
  1: Incorrect response, but remembered the context
  0: Complete blackout, did not recognize the card

Interval Logic:
  If rating q < 3 (Failed review, reset memory trace):
    r = 0
    I = 1
  If rating q >= 3 (Successful review):
    If r == 0:
      I = 1
    Else if r == 1:
      I = 6
    Else:
      I = round(previous_I * EF)
    
    // Update Easiness Factor
    EF = EF + (0.1 - (5 - q) * (0.08 + (5 - q) * 0.02))
    if EF < 1.3:
      EF = 1.3
    
    r = r + 1

Next Review Date:
  next_review_date = current_date + I


=============================================================================
3. DIFFICULTY ADAPTATION ENGINE
=============================================================================
Translates the student's BKT mastery score (0.0 to 1.0) into discrete content and
interaction bands:

  - Mastery [0.0 - 0.3] -> BEGINNER Band
    * Focus: Simplistic vocabulary, story-mode notes, basic flashcards (concept definitions)
    * Quizzes: Level 1 MCQs, generous hints, mascot encouragement
    * AI Tutor: Speaks with visual analogies (e.g. comparing cells to kitchens)

  - Mastery [0.3 - 0.6] -> DEVELOPING Band
    * Focus: Standard textbooks/notes, mixed flashcards (concept connections)
    * Quizzes: Level 2 MCQs + simple fill-in-the-blanks
    * AI Tutor: Scaffolded prompts (leads the student step-by-step)

  - Mastery [0.6 - 0.8] -> PROFICIENT Band
    * Focus: Detailed study material with extension concepts, hard flashcards (applications)
    * Quizzes: Level 3 MCQs + short written answers
    * AI Tutor: Socratic dialogue (challenges answers, asks "why")

  - Mastery [0.8 - 1.0] -> MASTERED Band
    * Focus: Challenge notes, cross-chapter links, real-world context analysis
    * Quizzes: Advanced reasoning questions, essay-style answers
    * AI Tutor: Poses advanced "what-if" hypotheses


=============================================================================
4. WEAKNESS DETECTION & PRIORITY SCORING
=============================================================================
Rank-orders chapters to suggest the most valuable learning actions daily.

Formula:
  Priority = (1 - mastery_score) * recency_weight * exam_proximity_weight

Weights:
  - recency_weight = 1 + (days_since_last_studied / 30)
    * Captures decay over time (max weight increases by 1 for each month of neglect)
  - exam_proximity_weight = 1 + max(0, (30 - days_to_exam) / 30)
    * Activates as exams approach (ramps up to a 2x modifier on the eve of the exam)


=============================================================================
5. LEARNING STYLE CLASSIFICATION
=============================================================================
Silently aggregates the student's active session time:
  - visual_score: Time spent looking at diagrams, maps, sketch canvas.
  - auditory_score: Time spent listening to audio explanations/TTS, talking to tutor.
  - reading_writing_score: Time spent reviewing text notes, writing flashcards.

Classification (evaluated every 10 sessions):
  - Highest score determines the style: Visual / Auditory / Reading-Writing.
  - Tailors default tab layout (e.g., auto-triggering TTS, sorting diagrams first).
